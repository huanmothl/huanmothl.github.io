异常的两种处理方式

1.throws_处理单个异常

格式：在方法参数和方法体之间位置上写： throws 异常

throws的解决方法是把异常向上抛

2.throws_处理多个异常

1.throws 后可跟多个异常

2.可跟这两个一场的共同的一个父类

3.如果不知道父类是啥，可以直接throws Exception

使用场景

就是有一些无法自动回收的对象，需要手动回收，则在使用完该对象之后无论结果如何都对其进行回收，把回收语句写在finally中。
![[Pasted image 20240727162916.png]]

抛异常注意事项

1.当父类抛出异常时，子类对父类重写的方法抛不抛异常都可以

2.当父类没有抛出异常时，子类对父类重写的方法不能抛出异常
![[Pasted image 20240727181723.png]]

try_catch和throws的使用时机

1.如果处理异常之后，还想让后续的代码正常执行，我们使用try…catch
2.如果方法之间是递进关系（调用），我们可以先throws，但是到了最后需要用try…catch做一个统一的异常处理

打印异常信息的三个方法

Throwable类中的方法：
String toString():输出异常类型和设置的异常信息
String getMessage():输出异常信息
void printStackTrace():打印异常信息是最全的：包括异常类型，信息，以及出现的行数等
