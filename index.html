线程和进程

进程：在内存中执行的应用程序
![[Pasted image 20240811092829.png|495]]
线程：进程中最小的执行单元
负责当前进程中程序的运行，一个进程中至少有一个线程，也可能会有多个线程

多线程：
1.软件中的耗时操作
2.所有的聊天软件
3.所有的后台服务器
提高cpu的利用率

并发和并行

并行：在同一个时刻，有多个指令在多个cpu线程上同时执行
![[Pasted image 20240811103850.png|435]]
![[Pasted image 20240811103859.png|435]]

单核cpu也能执行多个任务：
cpu会在多个任务之间进行高速切换

cpu调度

1.分时调度：指的是让所有的线程轮流获取CPU使用权,并且平均分配每个线程占用CPU的时间片
2.抢占式调度：多个线程轮流抢占CPU使用权，哪个线程先抢到了，哪个线程先执行，一般都是优先级高的先抢到CPU使用权的几率大，java程序就是抢占式的调度

主线程

是指CPU和内存之间专门为main方法开辟的线程

介绍

1.概述:就是基本类型对应的类(包装类),我们需要将基本类型转成包装类,从而让基本类型拥有类的特性(说白了,将基本类型转成包装类之后,就可以使用包装类中的方法操作数据)
2.为啥要学包装类:
a.将来有一些特定场景,特定操作,比如调用方法传递包装类比如:Arrayist集合,里面有一个方法add(Integer i),此时我们不能调用add方法之后直接传递基本类型,因为引用类型不能直接接收基本类型的值,就需要先将基本类型转成包装类,传递到add方法中
b.将来我们还可以将包装类转成基本类型:
包装类不能直接使用+ - * /,所以需要将包装类转成基本类型,才能使用 + - * /

基本类型 包装类 [[缓存对象]]
byte Byte -128~127
short Short -128~127
int Integer -128~127
long Long -128~127
float Float 没有

char Character 0~127
boolean Boolean true和false

Calendar日历类介绍

Date类中的大部分对象已被弃用，现在基本上用的是Calendar日历类，

Calendar日历类用法

字段值 含义
YEAR 年
MONTH 月（从0开始，可以+1使用）
DAY_OF_MONTH 月中的天（几号）
HOUR 时（12小时制）
HOUR_OF_DAY 时（24小时制）
MINUTE 分
SECOND 秒
DAY_OF_WEEK 周中的天（周几，周日为1，可以-1使用）
